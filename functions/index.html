Functions

Programming-speak for "verbs"

Some verbs we've already met: 

>>> int() # converts non-integers to integers
>>> str() # converts non-strings to strings
>>> len() # measures the length of a string, list, or dictionary

These are "built-in" functions, as they come with Python.

Functions do things;

Variables (integers, strings, lists) are things.

When you "run" a function,

you call it.

"Hey, len(), could you please measure this string for me?"

>>> len(this_string)
10

"Thanks, len(), you're the best."

The bit in the parentheses is the "argument".

>>> len(this_string)

The fun part:

You can make (define) your own functions!

Why would you want to define your own function?

"Function: A block of organized, reusable code that is used to perform a single, related action."

A function is a verb, so

"Computer, when I say, 'Measure and append this word,' what I mean is

  1. assign the length of the word to another variable,
  2. append the word to the end of a list,
  3. and tell me the word's length in plain language."

# word_measure.py
# gives the length of a word and adds it to a list
the_word = "Python" # create the word 
word_list = [] # make sure there's a list to append to
# assign the length of the word to another variable
word_length = len(the_word)
# append the word to the end of a list
word_list.append(the_word)
# and tell me the word's length in plain language
print "'" + the_word + "' is " + str(word_length) + " letters long."
 

$ ipython word_measure.py
Enter a word for me to measure. Python
'Python' is six letters long.

Do you want to have to type/paste this code in every time you need it?

In this case, we've created an "anonymous" function (no argument).
